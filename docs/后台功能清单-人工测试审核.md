# MacAgent 后台功能清单 · 人工测试审核

> 基于当前代码库整理，便于逐项人工测试与审核。每项含：功能说明、代码位置、关键参数、测试要点。

---

## 一、项目结构概览

```
MacAgent/
├── backend/                          # Python FastAPI 后端
│   ├── main.py                       # 入口、lifespan、路由注册
│   ├── app_state.py                  # 全局状态、TaskTracker、Feature Flags
│   ├── auth.py                       # 隧道认证
│   ├── connection_manager.py        # WebSocket 连接与广播
│   ├── ws_handler.py                 # WebSocket /ws 消息分发
│   ├── llm_config.py                 # LLM 配置持久化
│   ├── smtp_config.py / github_config.py
│   ├── data/                         # llm_config.json, smtp, github, contexts/
│   ├── routes/                       # HTTP 路由（见下文「HTTP API 一览」）
│   ├── agent/                        # Agent 核心、上下文、自愈、升级、EvoMap、Capsule
│   ├── tools/                        # 工具实现 + generated/
│   ├── llm/                          # tool_parser_v2, json_repair
│   └── runtime/                      # mac_adapter, base, registry
│
└── MacAgentApp/                      # macOS Swift/SwiftUI 客户端
    ├── ViewModels/AgentViewModel.swift
    ├── Services/BackendService.swift, ProcessManager.swift, TunnelManager.swift
    ├── Models/Message.swift
    └── Views/                        # ChatView, SettingsView, ToolPanelView, TunnelView...
```

## 二、启动与生命周期（main.py lifespan）

| 步骤 | 内容 | 位置/说明 |
|------|------|------------|
| 1 | 主 LLM 客户端 | `LLMClient()`，用户配置（DeepSeek/New API 等），用于 Chat |
| 2 | 云端 LLM | `cloud_llm_client`，自主任务选「远程」时专用 |
| 3 | 本地 LLM | `local_llm_client`（Ollama 11434，默认模型 qwen2.5-coder:7b） |
| 4 | 反思用 LLM | `reflect_llm`（Ollama，qwen2.5:7b） |
| 5 | Runtime 适配器 | `get_runtime_adapter()` → MacAdapter |
| 6 | AgentCore | `AgentCore(llm, runtime_adapter)`，注册工具、绑定 router |
| 7 | AutonomousAgent | `enable_reflection=True`, `enable_model_selection=True` |
| 8 | BGE 预加载 | `ENABLE_VECTOR_SEARCH=true` 时 `preload_embedding_model()` |
| 9 | Self-Upgrade 编排器 | 注入 `llm_chat`、`on_load_generated_tools`、`get_existing_tools` |
| 10 | Capsule 启动 | 异步 `bootstrap_capsules(run_sync_first=True)` |
| 11 | EvoMap | `ENABLE_EVOMAP=true` 时异步 `evomap_svc.initialize()` |
| 12 | EvoMap 升级钩子 | `init_evomap_upgrade_hook()` |
| 13 | EventBus 服务 | ErrorService、SelfHealingWorker、UpgradeService，绑定 broadcast |
| 14 | 系统消息服务 | `set_broadcast(connection_manager.broadcast_all)` |
| 15 | 启动日志分析 | 8 秒后 `log_analyzer.analyze_on_startup(log_buffer)` |

**审核要点**：启动无报错、日志中上述各步有对应输出；BGE/EvoMap 按环境变量生效。

## 三、Prompt 与 Token 优化（必审）

| # | 功能 | 位置 | 说明与参数 | 测试要点 |
|---|------|------|------------|----------|
| 3.1 | System Prompt 分级 | `agent/prompt_loader.py` | LITE（~400 字）用于简单对话，FULL 用于需工具的复杂任务；`get_system_prompt_for_query(query)` | 发「你好」→ LITE；发「打开 Safari」→ FULL；看日志 "Using LITE/FULL system prompt" |
| 3.2 | 查询复杂度分类 | `prompt_loader._classify_query_complexity()` | 短问候/闲聊→simple；含「打开、终端、截图、邮件…」→complex；问号结尾且无「怎么、如何、帮我」→simple | 多句典型话术验证 simple/complex 是否合理 |
| 3.3 | 进化规则追加 | `prompt_loader._load_evolved_rules()` | 从 `data/agent_evolved_rules.md` 读非 # 行，追加到 system prompt | 改 rules 后发一条对话，确认行为变化 |
| 3.4 | 上下文 token 上限 | `context_manager.ConversationContext` | `max_context_tokens=2000` | 长对话时上下文是否被压在此内 |
| 3.5 | 最近消息条数 | `context_manager` | `max_recent_messages=10` | 超 10 条后最老是否丢弃 |
| 3.6 | 向量检索上下文 | `context_manager.get_context_messages()` | `recent_count=4`，`semantic_count=3`，`max_tokens` 约束 | 开启向量时是否带相关历史；关闭时回退最近消息 |
| 3.7 | 工具 Schema 裁剪 | `agent/core.get_tool_schemas(query)`；`tools/registry.get_relevant_schemas()` | `max_tools=8`，`always_include=["terminal","file_operations","app_control"]`；BGE 语义或关键词回退 | 不同 query 下 "Tool schema pruned: x/总数" 是否合理；工具仍可调用 |
| 3.8 | LLM 生成长度 | `agent/llm_client.LLMConfig` | `max_tokens=4096` | 超长回复是否被截断 |
| 3.9 | Token 使用量 | `llm_client` stream 中 `chunk.usage`；前端 `TokenUsage` | prompt_tokens, completion_tokens, total_tokens | 对话结束后界面是否展示且与 API 一致 |
| 3.10 | 本地模型 System Prompt | `agent/local_tool_parser` | 本地模型用 `LOCAL_MODEL_SYSTEM_PROMPT` 等 | 切 Ollama/LM Studio 时 tool_calls 解析正常 |

## 使用说明

1. **必审**：先完成「三、Prompt 与 Token 优化」逐项打勾。
2. **核心**：再测「六、Agent 核心」「七、WebSocket」「八、任务跟踪」「九、自主 Agent」。
3. **工具**：按「十、工具列表」每类至少一次成功/失败用例。
4. **扩展**：按「十一～十三」做 Capsules、EvoMap、升级、自愈、系统消息与日志。
5. **接口**：按「十四」过一遍 HTTP API（可用 curl/Postman）。
6. **联调**：按「十六」做 Mac App 与后端联调（可选）。

建议在文档中为每项标注「通过 / 不通过 / 待测」，便于回归与排期。

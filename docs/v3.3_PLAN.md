# v3.3 功能清单

v3.3 在 v3.2 基础上完成「人机协同 + 安全增强 + 多智能体 + 会话持久化」四大方向，目标逼近 Claude Code 级体验。

**实施状态**：v3.3 实施中。

---

## 一、功能总览

| 方向 | 功能 | Flag | 状态 |
|------|------|------|------|
| 运维 | FeatureFlag 体系化（REST API + 热更新 + 持久化）| — | ✅ 已完成 |
| 鲁棒 | 幂等任务（同一 task hash 不重复执行）| `ENABLE_IDEMPOTENT_TASKS`（v3.2 预留）| ✅ 已完成 |
| 人机协同 | HITL 人工审批（关键动作需用户确认）| `ENABLE_HITL` | ✅ 已完成 |
| 安全 | 统一审计日志（全量操作记录 + 查询 API）| `ENABLE_AUDIT_LOG` | ✅ 已完成 |
| 会话 | Session Resume / Fork（会话持久化与恢复）| `ENABLE_SESSION_RESUME` | ✅ 已完成 |
| 扩展 | Subagent（任务拆分给子 Agent 并行执行）| `ENABLE_SUBAGENT`（v3.2 预留）| ✅ 已完成 |

## 二、新增 FeatureFlag

所有新 Flag 遵循 `MACAGENT_` 前缀，在 `backend/app_state.py` 中定义。

### 新增布尔开关

| Flag | 环境变量 | 默认值 | 说明 |
|------|----------|--------|------|
| `ENABLE_HITL` | `MACAGENT_ENABLE_HITL` | **False** | 关键动作需用户确认；关闭时所有动作自动执行 |
| `ENABLE_AUDIT_LOG` | `MACAGENT_ENABLE_AUDIT_LOG` | **True** | 全量操作审计日志，写入 `data/audit/` |
| `ENABLE_SESSION_RESUME` | `MACAGENT_ENABLE_SESSION_RESUME` | **False** | 会话持久化与恢复 / Fork |

### 新增数值配置

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|----------|--------|------|
| `HITL_CONFIRMATION_TIMEOUT` | `MACAGENT_HITL_CONFIRMATION_TIMEOUT` | **120**（秒）| 用户未确认的等待超时 |
| `IDEMPOTENT_CACHE_TTL` | `MACAGENT_IDEMPOTENT_CACHE_TTL` | **86400**（秒，24h）| 幂等缓存过期时间 |
| `SUBAGENT_MAX_CONCURRENT` | `MACAGENT_SUBAGENT_MAX_CONCURRENT` | **3** | 最大并行子 Agent 数 |
| `SUBAGENT_TIMEOUT` | `MACAGENT_SUBAGENT_TIMEOUT` | **300**（秒）| 单个子 Agent 超时 |
| `AUDIT_LOG_MAX_SIZE_MB` | `MACAGENT_AUDIT_LOG_MAX_SIZE_MB` | **100** | 审计日志最大磁盘占用 |

## 三、功能详解

### 3.1 FeatureFlag 体系化

**文件**：`backend/routes/feature_flags.py`（新建）、`backend/services/feature_flag_service.py`（新建）

#### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/feature-flags` | GET | 返回所有 Flag 当前值、默认值、来源（env/config/default）|
| `/feature-flags` | PATCH | 热更新指定 Flag（不需重启）|
| `/feature-flags/reset` | POST | 重置所有 Flag 为默认值 |

#### 实现要点

- 持久化到 `data/config/feature_flags.json`
- 优先级：运行时热更新 > `feature_flags.json` > 环境变量 > 代码默认值
- 热更新通过修改 `app_state` 模块级变量实现，无需重启
- Flag 元数据包含：名称、描述、类型（bool/int/float）、默认值、当前值、来源

### 3.2 幂等任务

**文件**：`backend/services/idempotent_service.py`（新建）

#### 设计

- 缓存键：`sha256(action_type + sorted(params))` 的前 16 字节 hex
- 缓存位置：`data/task_cache/{hash}.json`
- 记录内容：`{hash, action_type, params_hash, result, created_at, expires_at}`
- 集成点：`AutonomousAgent._execute_action()` 安全检查后、执行前

#### 关键规则

- 只缓存**成功**的执行结果
- `run_shell`、`create_and_run_script` **不缓存**（命令有副作用）
- `read_file`、`call_tool(search)` 等只读操作缓存
- TTL 可配置，默认 24h

### 3.3 HITL（人工审批）

**文件**：`backend/services/hitl_service.py`（新建）、`backend/routes/hitl.py`（新建）

#### 需要确认的动作类型

| 风险等级 | 动作类型 | 需确认条件 |
|---------|---------|-----------|
| 🔴 高危 | `run_shell`（含 sudo/rm/chmod）| 始终 |
| 🟠 中危 | `delete_file`、`move_file` | 始终 |
| 🟡 低危 | `write_file`（系统路径附近）| 路径匹配 |
| 🟢 安全 | `read_file`、信息查询 | 无需确认 |

#### 流程

```
Agent 产生动作
  → HITL 检查 (hitl_service.should_require_confirmation)
    → 需确认:
      1. 写入审计日志 (type=hitl_pending)
      2. 广播确认请求到 WebSocket
      3. 等待用户响应 (asyncio.Event, timeout=120s)
      4. 用户确认 → 执行 → 写审计日志 (type=hitl_approved)
      5. 用户拒绝 → 跳过 → 写审计日志 (type=hitl_rejected)
      6. 超时 → 跳过 → 写审计日志 (type=hitl_timeout)
    → 不需确认: 直接执行
```

#### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/hitl/pending` | GET | 查询当前等待确认的动作列表 |
| `/hitl/confirm/{action_id}` | POST | 确认执行某动作 |
| `/hitl/reject/{action_id}` | POST | 拒绝执行某动作 |

### 3.4 统一审计日志

**文件**：`backend/services/audit_service.py`（新建）、`backend/routes/audit.py`（新建）

#### 审计事件类型

| 类型 | 说明 |
|------|------|
| `action_execute` | 动作执行（命令、文件操作）|
| `action_blocked` | 安全拦截 |
| `hitl_pending` | 等待用户确认 |
| `hitl_approved` | 用户批准 |
| `hitl_rejected` | 用户拒绝 |
| `hitl_timeout` | 确认超时 |
| `session_start` | 会话开始 |
| `session_end` | 会话结束 |
| `config_change` | 配置变更 |
| `error` | 错误事件 |

#### 日志格式

```json
{
  "id": "uuid",
  "ts": "ISO-8601",
  "type": "action_execute",
  "task_id": "...",
  "session_id": "...",
  "action_type": "run_shell",
  "params_summary": "ls -la /tmp",
  "result": "success",
  "details": {},
  "risk_level": "low"
}
```

#### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/audit` | GET | 查询审计日志（支持 `type`、`task_id`、`session_id`、`offset`、`limit` 过滤）|
| `/audit/stats` | GET | 审计统计摘要 |
| `/audit/{log_id}` | GET | 获取单条审计记录详情 |

#### 存储

- 文件：`data/audit/{date}.jsonl`（按日分片）
- 自动清理：超过 `AUDIT_LOG_MAX_SIZE_MB` 时删除最早的日志文件
- 与 Trace 系统互补：Trace 记录推理过程，Audit 记录操作事实

### 3.5 Session Resume / Fork

**文件**：`backend/services/session_service.py`（新建）、`backend/routes/sessions.py`（新建）

#### Checkpoint 格式扩展

在 `TaskCheckpoint` 基础上新增：

| 字段 | 类型 | 说明 |
|------|------|------|
| `memory_snapshot` | `Dict` | episodic memory 快照 |
| `context_summary` | `str` | 当前上下文压缩摘要 |
| `plan_state` | `Dict` | Plan-and-Execute 进度 |
| `fork_parent_id` | `str?` | Fork 来源的 session ID |

#### Resume 流程

```
1. GET /sessions/{session_id}/checkpoints → 列出可恢复的检查点
2. POST /sessions/{session_id}/resume?checkpoint_id=xxx
   → 加载 TaskCheckpoint
   → 恢复 memory 快照与上下文
   → 从 checkpoint.current_iteration 继续执行
```

#### Fork 流程

```
1. POST /sessions/{session_id}/fork?checkpoint_id=xxx
   → 创建新 session_id
   → 深拷贝 checkpoint（fork_parent_id = 原 session_id）
   → 返回新 session_id，客户端可独立操作
```

#### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/sessions` | GET | 列出所有可恢复的会话 |
| `/sessions/{session_id}/checkpoints` | GET | 列出会话的检查点 |
| `/sessions/{session_id}/resume` | POST | 恢复会话执行 |
| `/sessions/{session_id}/fork` | POST | 从检查点分支新会话 |

### 3.6 Subagent（子 Agent 并行执行）

**文件**：`backend/services/subagent_service.py`（新建）、`backend/routes/subagents.py`（新建）

#### 架构

```
主 Agent (AutonomousAgent)
  ├─ 任务分析 → 判断是否需要拆分
  ├─ SubagentScheduler.split_task(task_description)
  │   → [SubTask1, SubTask2, SubTask3]
  ├─ SubagentPool.execute_all(subtasks)
  │   ├─ Worker1: 轻量 AgentCore 实例
  │   ├─ Worker2: 轻量 AgentCore 实例
  │   └─ Worker3: 轻量 AgentCore 实例
  └─ 结果聚合 → 写入主 Trace
```

#### 关键约束

- 子 Agent **共享** LLM client（避免重复初始化）
- 子 Agent **独有** 工具沙箱（独立 cwd、独立 trace）
- 最大并行数可配置（`SUBAGENT_MAX_CONCURRENT`，默认 3）
- 单个子 Agent 超时可配置（`SUBAGENT_TIMEOUT`，默认 300s）
- 子 Agent 的 Trace 关联到主任务（parent_task_id）

#### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/subagents` | GET | 查看当前活跃的子 Agent |
| `/subagents/{task_id}` | GET | 查看任务的子 Agent 状态 |

## 四、与 v3.2 的关系

v3.3 不破坏 v3.2 任何接口或行为。所有新功能均通过 FeatureFlag 控制，可独立关闭退回 v3.2 行为：

```bash
# 回退 v3.3（如需排查）
export MACAGENT_ENABLE_HITL=false
export MACAGENT_ENABLE_AUDIT_LOG=false
export MACAGENT_ENABLE_SESSION_RESUME=false
export MACAGENT_ENABLE_SUBAGENT=false
export MACAGENT_ENABLE_IDEMPOTENT_TASKS=false
```

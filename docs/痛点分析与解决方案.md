# MacAgent 痛点分析与解决方案

> 目标：打造一个真正可用的 AI 助手，解决 LLM 执行能力不足、文件关联弱、终端/输入控制不完善等问题。

---

## 一、现状与根因

### 1. Skill/Capsule 路径获取能力弱

**现象**：LLM 在执行任务时没有很好利用技能库，不会主动查找或执行 Capsule。

**根因分析**：

| 问题 | 位置 | 说明 |
|------|------|------|
| **capsule 被裁剪** | `core.py:122` | `always_include` 只有 `["terminal", "file_operations", "app_control"]`，**capsule 不在其中**。当工具数 > 8 时，capsule 可能被语义裁剪掉，LLM 根本看不到 capsule 工具 |
| 推荐逻辑未注入 | `core.py` run_stream | 只有 `autonomous_agent` 有 capsule 扫描注入，**Chat 主流程没有**在 system prompt 中注入「匹配的 capsule 列表」 |
| Prompt 过于简略 | `capsule.md` | 仅 4 行，缺少「何时先查 capsule」「如何 find→execute」的明确指引 |

### 2. 不会检查文件

**现象**：直接操作文件而不先 read/check，导致覆盖、误删或重复创建。

**根因**：

- `behavior.md` 有「文件已存在 → 先 read 判断」规则，但不够突出
- 没有在 tools 描述中强调「操作前先 read 或 list」
- 缺少「智能推断路径」：用户说「那个文件」时，LLM 不知道指哪个

### 3. 终端使用不够智能

**现象**：不会合理选择 terminal vs 其他工具，长期进程超时、命令格式不当。

**根因**：

- 已有 `background: true` 规则，但可能被忽略
- 无交互式终端会话，只能单命令执行
- 无命令历史/上下文，多步操作需每次重新描述

### 4. 键盘鼠标模拟不完善

**现象**：GUI 自动化能力弱，依赖坐标、无法识别界面元素。

**根因**：

- 依赖 `cliclick` 外部工具，环境不一致易失败
- 无 OCR/元素检测，无法「点击某个按钮」
- 无录制回放，无法复用用户操作序列

### 5. 不能像 Cursor 做文件强关联

**现象**：无法像 Cursor 那样「当前打开的文件自动进入上下文」。

**根因**：

- 只有 `created_files` 记录，无 workspace 文件索引
- 无「当前打开文件列表」接口
- 无文件内容自动注入到 prompt

---

## 二、解决方案（按优先级）

### P0：立即见效（改配置/小改代码）

#### 1. 将 capsule 加入 always_include

**文件**：`backend/agent/core.py`

```python
# 修改 get_tool_schemas 中的 always_include
always_include=["terminal", "file_operations", "app_control", "capsule"],
```

**效果**：capsule 工具不再被裁剪，LLM 始终能看到并调用。

---

#### 2. 在 Chat 流程中注入匹配的 Capsule 提示

**思路**：在构建 system prompt 时，用 `find_capsule_by_task` 查询用户 query，若有匹配则注入「推荐技能」到 prompt。

**位置**：`prompt_loader.py` 的 `get_system_prompt_for_query`，或 `core.py` 构建 messages 前。

**伪代码**：

```python
# 在 get_system_prompt_for_query 或 core 中
from agent.capsule_registry import get_capsule_registry
reg = get_capsule_registry()
caps = reg.find_capsule_by_task(query, limit=3, min_score=0.6)
if caps:
    hint = "\n\n[推荐技能（优先调用 capsule 执行）：\n"
    for c in caps:
        hint += f"- {c.id}: {c.name or c.description}\n"
    hint += "]\n"
    system_prompt += hint
```

**效果**：LLM 明确知道有哪些技能可用，提高调用率。

---

#### 3. 强化「先检查再操作」的 Prompt

**文件**：`backend/data/prompts/behavior.md` 或 `tools.md`

增加：

```markdown
## 文件操作前必读
- 修改/覆盖前：先 file_operations read 确认内容
- 删除前：先 file_operations info 确认路径正确
- 用户说「那个文件」「刚才的」：从 created_files 或对话历史推断路径
```

---

### P1：短期增强（1–2 周）

#### 4. 文件强关联：Workspace 上下文

**设计**：

- 新增 `WorkspaceContext`：记录「当前工作目录」「用户最近打开/编辑的文件」
- Mac 端通过 AppleScript/API 获取「Finder 当前目录」「VS Code/Cursor 打开的文件」
- 在 system prompt 中注入：`[当前工作区：/path, 打开的文件：a.py, b.md]`

**实现要点**：

- 新增 `agent/workspace_context.py`
- MacAgentApp 通过 MCP 或 HTTP 上报「打开的文件」
- `get_system_prompt_for_query` 中合并 workspace 信息

---

#### 5. 终端增强：交互式会话 + 历史

- 维护 `TerminalSession`：同一 session 内复用 cwd、环境变量
- 支持「上一条命令的输出」作为下一条的输入（通过 context 注入）
- 可选：流式输出，避免长命令超时误判

---

#### 6. 扩充 capsule.md 指引

**文件**：`backend/data/prompts/capsule.md`

```markdown
# 技能 Capsule

## 何时使用
- 用户任务涉及「技能」「自动化」「批量处理」「特定场景」时，先 capsule find 再执行
- 系统推荐匹配 Capsule 时，直接调用 capsule execute

## 调用流程
1. capsule find(task="用户任务关键词") → 获取匹配列表
2. 若有匹配，capsule execute(capsule_id="...", inputs={...})
3. 指令型(instruction_mode=true)：按返回的步骤，用 file_operations/terminal 等逐步完成

## 禁止
- 不要跳过 find 直接猜测 capsule_id
- 不要用 terminal 执行本应由 capsule 封装的操作
```

### P2：中期能力（1–2 月）

#### 7. 输入控制增强：元素级自动化

- 集成 **Apple Accessibility API** 或 **pyautogui + 图像识别**：通过控件类型、名称定位
- 可选：OCR（如 pytesseract）识别屏幕文字，实现「点击包含 X 的按钮」
- 录制回放：记录用户操作序列 → 生成可复用的 input_control 调用序列

---

#### 8. 文件内容索引

- 对 workspace 内文件做轻量索引（路径、修改时间、前 N 行）
- 用户说「那个配置文件」时，通过向量检索定位
- 大文件：只索引摘要或 chunk，按需 read

## 四、与 Cursor 的差异（为何难以完全对齐）

| 能力 | Cursor | MacAgent |
|------|--------|----------|
| 文件关联 | IDE 内置，打开即 context | 需额外上报/索引 |
| 代码理解 | 全项目解析、符号索引 | 依赖 read + LLM 理解 |
| 终端 | 集成 Terminal，可复用 shell 状态 | 单命令执行，无持久 shell |
| 安全边界 | 沙箱、用户确认 | 需 execution_guard、权限控制 |

MacAgent 的定位是「系统级助手」，不是「代码 IDE」。可借鉴 Cursor 的「强关联」思路，通过 workspace 上报 + 文件索引 来逼近，但不必完全一致。

---

## 五、快速验证清单

修改后可用以下用例自测：

1. **Capsule**：用户说「帮我检查系统状态」→ 应调用 capsule（若有对应技能）
2. **文件检查**：用户说「修改 config.json 里的 port」→ 应先 read 再 write
3. **终端**：用户说「启动 Flask 开发服务器」→ 应使用 `background: true`
4. **追问**：用户说「项目在哪」→ 应直接引用 created_files，不重新执行
5. **终端会话**：先「cd /tmp」再「ls」→ 第二条命令应复用 /tmp 作为 cwd

## 六、MacAgentApp 集成 Workspace 上报

为启用文件强关联，MacAgentApp 需在适当时机调用 Workspace API 上报当前工作区：

```http
POST /workspace
Content-Type: application/json

{
  "session_id": "当前会话ID（与 Chat 的 conversation_id 一致）",
  "cwd": "/Users/xxx/project",
  "open_files": ["/Users/xxx/project/src/main.py", "/Users/xxx/project/README.md"]
}
```

**上报时机建议**：
- 用户切换工作目录时（如 Finder 当前窗口、终端 cwd）
- 用户打开/切换文件时（如 VS Code/Cursor 的 active editor）
- 会话开始时，若可获取当前上下文

# Agent Live — AI 执行可视化模块设计

> 让用户直观看到 AI 在执行、在动、在用工具、在用 MCP 的有趣视觉效果

---

## 一、设计目标

- **直观**：一眼看懂「AI 在干什么」
- **有趣**：有动效、有反馈、有「活着」的感觉
- **信息密度适中**：不喧宾夺主，但能传达关键状态

---

## 二、核心概念：Agent Pulse（智能体脉动）

将 Agent 执行过程抽象为**中心节点 + 外围能力圈**的实时动画：

```
                    ┌─────────────┐
                    │   MCP 🌐   │  ← 外部能力（GitHub、Brave、Memory…）
                    └──────┬──────┘
                           │ 数据流
              ┌────────────┼────────────┐
              │            ▼            │
         ┌────┴────┐  ┌─────────┐  ┌────┴────┐
         │ 工具 🔧 │  │   AI 🧠  │  │ 工具 🔧 │  ← 内置工具
         └────┬────┘  │ 思考中  │  └────┬────┘
              │       └────┬────┘       │
              │            │            │
              └────────────┼────────────┘
                           ▼
                    ┌─────────────┐
                    │  输出 📤   │
                    └─────────────┘
```

---

## 三、视觉方案（三选一或组合）

### 方案 A：轨道 + 节点（Orbital）

- **中心**：AI 大脑图标，思考时脉动发光（紫色）
- **内圈**：常用工具图标（file、terminal、browser…）沿轨道排布，被调用时高亮 + 脉冲
- **外圈**：MCP 服务节点（GitHub、Brave、Memory…），连接时显示「数据流」动画
- **动效**：粒子从中心流向被调用的工具，再流回中心

**适合**：偏「系统架构」感，信息量大

---

### 方案 B：流水线（Pipeline）

- **横向流程**：`输入 → 思考 → 工具/MCP → 输出`
- **每个阶段**：图标 + 状态（idle / active / done）
- **当前阶段**：放大、发光、粒子环绕
- **工具调用**：工具图标从「工具池」飞出，落到流水线，执行完飞回

**适合**：偏「流程」感，易理解

---

### 方案 C：神经网（Neural）

- **背景**：网格或六边形蜂窝
- **节点**：AI、工具、MCP 为不同颜色节点
- **连线**：节点间有虚线，激活时变实线 + 光点流动
- **思考**：中心节点「呼吸」动画
- **工具调用**：从中心向工具节点发射光波

**适合**：偏「科技感」，与现有 Cyberpunk 风格一致

---

## 四、推荐：方案 C 变体 —「Agent Neural Live」

结合现有 Cyberpunk 风格，做**神经网 + 轨道**混合：

| 区域 | 视觉 | 数据来源 |
|------|------|----------|
| 中心 | AI 大脑，思考时紫色脉动 | `isStreamingLLM` |
| 内环 | 6–8 个常用工具图标，调用时高亮 | `recentToolCalls` / `actionLogs` |
| 外环 | MCP 服务节点（可配置显示） | `/mcp/servers` + 调用记录 |
| 连线 | 中心→工具、中心→MCP 的光点流动 | 实时事件 |
| 底部 | 当前动作文字（如「正在执行 file_read」） | `action_plan` / `action_executing` |

## 五、技术实现要点

### 5.1 数据源

- `MonitoringViewModel` 已有：`actionLogs`、`recentToolCalls`、`isStreamingLLM`、`currentTaskData`
- 新增：`lastToolCall`、`lastMcpCall`（最近一次调用的工具/MCP 名）
- 可选：轮询 `/mcp/tools` 或通过 `monitor_event` 解析 MCP 调用

### 5.2 动效实现

- **SwiftUI + TimelineView**：定时驱动动画
- **Canvas / Shape**：绘制连线、光点
- **SymbolEffect**：SF Symbol 的 `.bounce`、`.pulse` 等
- **Metal**：若需更复杂粒子，可用现有 `CyberGrid.metal`

### 5.3 模块入口

- 在 `MonitoringWindowView` 增加新 Tab：**LIVE**（或「脉动」）
- 独立 View：`AgentLiveView.swift`

---

## 六、MVP 功能清单

1. **中心 AI 节点**：根据 `isStreamingLLM` 切换 idle / thinking 动画
2. **工具环**：展示 6 个常用工具图标，最近调用者高亮 2 秒
3. **底部状态条**：显示当前动作类型（如「file_read」「terminal」）
4. **简单连线**：中心到高亮工具的光线动画

后续可扩展：MCP 节点、粒子流、声音反馈等。

---

## 七、与现有模块关系

| 现有 Tab | 新 Tab LIVE |
|----------|-------------|
| EXEC：时间轴、步骤列表 | LIVE：图形化、实时脉动 |
| 信息密度高、适合复盘 | 信息密度低、适合「看着爽」 |

两者互补：EXEC 负责「细节」，LIVE 负责「氛围」。

---

## 八、实现状态（v3 — Cyberdeck Terminal）

**单一设计**：赛博朋克 2077 风格「神经接口终端」，不再提供多模式切换。

### 8.1 视觉元素

| 元素 | 说明 |
|------|------|
| 背景网格 | 淡青色网格线，模拟全息投影 |
| 雷达扫描 | 中心扇形持续旋转，类似声呐 |
| 扫描线 | 水平光带缓慢下移，CRT 感 |
| 角标 | 四角 L 形装饰，2077 风格 |
| 边框 | 思考时紫色、执行时青色发光 |

### 8.2 终端内容

- **待命**：`> STANDBY` + 闪烁光标
- **思考**：`> PROCESSING...` 动态省略号 + 紫色高亮
- **执行**：`> EXECUTE: RUN_SHELL` + 参数 + 终端输出（若有）

### 8.3 参考

- Cyberpunk 2077: 全息 HUD、AR 叠加、霓虹边框
- 终端：命令执行、输出流、光标闪烁

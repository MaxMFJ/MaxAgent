# MacAgent 打包技术路线：后台内置与启动服务

> 已实施：后台（backend）内置到 Mac App Bundle，支持应用内「启动服务开关」控制后台启停。

---

## 一、当前架构（已实施）

### 1.1 目录结构（打包后）

```
MacAgentApp.app/
├── Contents/
│   ├── MacOS/
│   │   └── MacAgentApp
│   ├── Resources/
│   │   ├── backend/              ← 内置后台（Build Phase 复制）
│   │   │   ├── main.py
│   │   │   ├── start.sh
│   │   │   ├── paths.py           ← 统一路径，支持 MACAGENT_DATA_DIR
│   │   │   ├── requirements.txt
│   │   │   ├── agent/
│   │   │   ├── tools/
│   │   │   ├── routes/
│   │   │   ├── data/              ← 模板（只读），可写数据在 Application Support
│   │   │   └── ...
│   │   └── Assets.car
│   └── Info.plist
```

### 1.2 路径解析逻辑（ProcessManager）

| 优先级 | 路径 | 场景 |
|--------|------|------|
| 1 | `Bundle.main.resourcePath + "/backend"` | 打包后运行 |
| 2 | `currentDirectoryPath + "/../backend"` | 工作目录为项目根时 |
| 3 | `~/Desktop/未命名文件夹/MacAgent/backend` 等 | 从 Xcode 运行（DerivedData）时 |
| 4 | 与 .app 同级的 `../backend`（向上多级） | 其他场景 |
| 5 | 兜底 | `resourcePath + "/backend"` |

**从 Xcode 运行**：若 backend 未被打包进 DerivedData，需在 Scheme 中设置 **Working Directory** 为项目目录（如 `MacAgent/MacAgentApp`），或确保 backend 在 `~/Desktop/未命名文件夹/MacAgent/backend`。

### 1.3 可写数据目录（Application Support）

- **Bundle 内 `data/` 只读**：打包后无法写入
- **可写目录**：`~/Library/Application Support/com.macagent.app/backend_data/`
- **逻辑**：`ProcessManager.getWritableDataDir()` 检测 bundle data 不可写时，使用 Application Support；首次启动从 bundle 复制 data 模板
- **环境变量**：启动 backend 时传入 `MACAGENT_DATA_DIR`，Python 侧 `paths.py` 统一读取

## 二、启动/关闭逻辑

### 2.1 启动流程

1. 用户点击「启动」或工具栏后端状态指示器
2. `ProcessManager.startBackend()` 调用
3. `getBackendPath()` 解析 backend 路径
4. `getWritableDataDir()` 检测 bundle data 可写性，若不可写则使用 Application Support 并传入 `MACAGENT_DATA_DIR`
5. 执行 `cd backend && bash start.sh`，环境变量传入子进程
6. `start.sh` 创建 venv、安装依赖、启动 `python main.py`

### 2.2 关闭流程

1. 用户点击「停止」
2. `ProcessManager.stopBackend()` 调用
3. 若由本 App 启动：`terminate()` 子进程 + `lsof -ti:8765 | xargs kill -9` 清理端口
4. 若由外部启动：仅终止本 App 启动的 Process，不杀端口

### 2.3 Xcode 配置

- **Copy Backend to Resources**（Run Script）：在 Link 之后、Resources 之前执行，从 `PROJECT_DIR/../backend` 复制到 `Contents/Resources/backend`。支持多路径回退（`PROJECT_DIR`、`SRCROOT`、`WORKSPACE_DIR`）。
- **Strip venv and sensitive data from backend**：Copy Backend 之后执行，删除 `venv`、`__pycache__`、`.git`，并过滤 `backend/data/` 中的敏感信息（见下表）。
- **项目结构要求**：backend 需与 MacAgentApp 同级，即 `MacAgent/backend/` 与 `MacAgent/MacAgentApp/`。
- **Archive 说明**：请使用 **MacAgentApp.xcworkspace**（在 MacAgentApp 目录下）进行 Archive，确保 CocoaPods 与 Copy Backend 脚本正确执行。
- **Debug 构建**：Copy Backend / Pre-install / Strip 均**仅 Archive 时执行**。Debug 直接使用项目目录 `backend/`（含 venv），不打包进 App，构建快、调试方便。需先 `cd backend && python3 -m venv venv && pip install -r requirements.txt`。

### 2.4 打包时过滤的敏感数据（backend/data/）

| 类型 | 路径 |
|------|------|
| 配置 | `agent_config.json` |
| 运行时数据 | `contexts/*.json`、`episodes/*.json`、`evomap/`、`model_selection/` |
| 其他 | `strategy_db.json`、`execution_guard_metrics.jsonl`、`query_classifier_metrics.jsonl`、`signatures.json`、`agent_evolved_rules.md` |

> 注意：`llm_config.json`、`github_config.json`、`smtp_config.json` 等若含密钥，建议在首次运行或配置界面中由用户填写，或另行排除。

## 三、Python 侧路径配置

### 3.1 paths.py

```python
# backend/paths.py
_BACKEND_DIR = os.path.dirname(os.path.abspath(__file__))
DATA_DIR = os.environ.get("MACAGENT_DATA_DIR") or os.path.join(_BACKEND_DIR, "data")
BACKEND_ROOT = _BACKEND_DIR
```

## 四、系统要求

- **macOS 14.0+**
- **Python 3.8+**（推荐 3.10+，系统 Python，`start.sh` 使用 `python3`）
- **打包体积**：~100MB（仅核心依赖），RAG/向量搜索（~600MB）在用户首次启用时自动安装到 Application Support
- **打包时**：Pre-install 仅安装 `requirements-core.txt`（fastapi、openai 等），不含 torch/sentence-transformers
- **RAG 启用**：打包版默认关闭。用户添加 `ENABLE_VECTOR_SEARCH=true` 到 `~/Library/Application Support/com.macagent.app/backend_data/.env` 后重启后端，首次启动会自动安装 RAG 依赖（0 操作）

## 六、降低用户部署难度

当前内置后台仍依赖本机 Python + venv + pip install，对非技术用户不友好。若需降低「后台启动」的部署门槛，可参考 [新环境部署指南](./新环境部署指南.md) 中的「降低后台部署难度：可选方案」（内嵌 Python、PyInstaller、一键脚本、远程后端等）。

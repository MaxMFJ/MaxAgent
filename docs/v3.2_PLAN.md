# v3.2 功能清单

v3.2 在 v3.1 基础上完成「可观测 + 可评估 + Phase C 研究增强」三大方向，同时随带交付了原本规划在 v3.3/Phase C 的能力。

**实施状态**：v3.2 已全部落地，31/31 单元测试通过（`backend/tests/test_v32.py`）。

## 二、FeatureFlag 完整说明

所有 Flag 在 `backend/app_state.py` 中定义，通过环境变量覆盖（前缀 `MACAGENT_`）。

### 自动开启（默认 True）

| Flag | 环境变量 | 默认值 | 说明 |
|------|----------|--------|------|
| `TRACE_TOKEN_STATS` | `MACAGENT_TRACE_TOKEN_STATS` | **True** | 每个 span 自动记录 token 消耗（prompt/completion/total） |
| `TRACE_TOOL_CALLS` | `MACAGENT_TRACE_TOOL_CALLS` | **True** | 工具调用结果自动写入 trace span |
| `ENABLE_IMPORTANCE_WEIGHTED_MEMORY` | `MACAGENT_ENABLE_IMPORTANCE_WEIGHTED_MEMORY` | **True** | Phase C：新增 episode 自动计算重要性分数；搜索时融合相似度与重要性 |
| `ENABLE_FAILURE_TYPE_REFLECTION` | `MACAGENT_ENABLE_FAILURE_TYPE_REFLECTION` | **True** | Phase C：反思时自动分类失败类型，选用专用 prompt 模板 |
| `ENABLE_ON_DEMAND_SKILL_FETCH` | `MACAGENT_ENABLE_ON_DEMAND_SKILL_FETCH` | **True** | **按需拉取 Skill**：启动时不全量同步 GitHub，仅在 LLM 需要某技能时才下载 |

### 手动开启（默认 False）

| Flag | 环境变量 | 默认值 | 说明 |
|------|----------|--------|------|
| `ENABLE_EXTENDED_THINKING` | `MACAGENT_ENABLE_EXTENDED_THINKING` | **False** | Phase C：启用 Extended Thinking（Anthropic provider）或 CoT 前缀（其他 provider） |
| `ENABLE_IDEMPOTENT_TASKS` | `MACAGENT_ENABLE_IDEMPOTENT_TASKS` | **False** | 幂等任务（实验性，v3.3 完整实现）|
| `ENABLE_SUBAGENT` | `MACAGENT_ENABLE_SUBAGENT` | **False** | 子 Agent 调度（v3.3 规划中，Flag 已预留）|

### 数值配置项

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|----------|--------|------|
| `HEALTH_DEEP_LLM_TIMEOUT` | `MACAGENT_HEALTH_DEEP_LLM_TIMEOUT` | **5.0**（秒）| `/health/deep` LLM 连通性探针超时 |
| `EXTENDED_THINKING_BUDGET_TOKENS` | `MACAGENT_EXTENDED_THINKING_BUDGET_TOKENS` | **8000** | Extended Thinking token 预算 |
| `ON_DEMAND_SKILL_FETCH_TIMEOUT` | `MACAGENT_ON_DEMAND_SKILL_FETCH_TIMEOUT` | **10.0**（秒）| 按需拉取单个 SKILL.md 的超时时间 |
| `ON_DEMAND_SKILL_MAX_FETCH` | `MACAGENT_ON_DEMAND_SKILL_MAX_FETCH` | **3** | 单次按需拉取最多获取几个新技能 |

## 三、功能详解

### 3.1 Trace 完善

**文件**：`backend/core/trace_logger.py`

新增 4 个函数：

| 函数 | 说明 |
|------|------|
| `get_trace_summary(task_id)` | 返回：span 数、总 token（prompt/completion/total）、平均延迟、工具调用次数、成功/失败数 |
| `list_traces(limit=50)` | 列出所有 task_id 及最新修改时间，按时间倒序 |
| `get_trace_spans(task_id, offset, limit, span_type)` | 分页获取 spans，支持按类型（llm/tool/error）过滤 |
| `delete_trace(task_id)` | 删除指定 task 的 trace 文件 |

`append_span()` 增强：自动注入 `ts`（ISO 时间戳）若 span 未提供。

### 3.2 Traces REST API

**文件**：`backend/routes/traces.py`

已注册到路由（`routes/__init__.py`）。

| 端点 | 方法 | 说明 |
|------|------|------|
| `/traces` | GET | 列出所有 traces（`limit` 参数，默认 50）|
| `/traces/{task_id}` | GET | 获取指定任务的 trace summary |
| `/traces/{task_id}/spans` | GET | 分页获取 spans（`offset`/`limit`/`span_type`）|
| `/traces/{task_id}` | DELETE | 删除指定任务的 trace 文件 |

### 3.3 /health/deep 深度健康检查

**文件**：`backend/routes/health.py`，端点 `GET /health/deep`

检查 8 个子系统：

| 子系统 | 检查内容 | 是否 required |
|--------|---------|---------------|
| `llm` | 向 LLM 发送 ping 请求，`HEALTH_DEEP_LLM_TIMEOUT` 超时控制 | ✅ 是 |
| `disk` | `shutil.disk_usage`，可用空间 > 500 MB | ✅ 是 |
| `memory` | psutil 内存使用率 < 95% | ✅ 是 |
| `vector_db` | 调用向量库 ping/load 验证 | 否 |
| `tool_router` | 检查工具路由是否初始化 | ✅ 是 |
| `task_tracker` | TaskTracker 内存任务数 | ✅ 是 |
| `traces_dir` | traces 目录是否可读写 | 否 |
| `evomap` | EvoMap 服务连通性 | 否 |

响应格式：
```json
{
  "healthy": true,
  "required_failed": [],
  "checks": { "llm": {"ok": true}, ... },
  "check_duration_ms": 123.4
}
```

`healthy` = 所有 required 子系统均通过。

### 3.4 Phase C — 失败分类反思

**文件**：`backend/agent/reflect_engine.py`

新增 `FailureType`（7 种）：

| 值 | 关键词触发 | 专用 Prompt 指引 |
|----|-----------|-----------------|
| `COMMAND_ERROR` | command not found, exit code, syntax error | 检查命令拼写/路径/依赖 |
| `PERMISSION` | permission denied, access denied, unauthorized | 检查权限/sudo/文件所有者 |
| `FILE_NOT_FOUND` | no such file, file not found, cannot open | 检查文件路径/前置步骤 |
| `NETWORK_ERROR` | connection refused, timeout, network unreachable | 检查网络/服务是否运行 |
| `TIMEOUT` | timed out, deadline exceeded, took too long | 分解任务/延长超时 |
| `LOGIC_ERROR` | assertion error, unexpected result, wrong output | 检查算法/数据流/边界 |
| `UNKNOWN` | （兜底）| 通用反思策略 |

`ReflectResult` 新增字段：`failure_type`、`root_cause`、`correct_approach`。

**开关**：`ENABLE_FAILURE_TYPE_REFLECTION=true`（默认启用）。

### 3.5 Phase C — 重要性加权 Memory

**文件**：`backend/agent/episodic_memory.py`

`Episode` 新增 `importance_score: float`（0-1），由 4 维度加权计算：

| 维度 | 权重 | 说明 |
|------|------|------|
| success | 35% | 成功任务权重更高 |
| complexity | 25% | 动作步骤越多，重要性越高（上限 `min(total_actions/10, 1.0)`）|
| feedback | 20% | 用户正向反馈加分 |
| recency | 20% | 指数衰减，半衰期 14 天 |

`search_similar()` 新增 `importance_weight=0.3` 参数：
```
final_score = (1 - importance_weight) * similarity + importance_weight * importance_score
```

`add_episode()` 自动调用 `compute_importance_score()`（当 Flag 开启时）。

`update_importance(episode_id, delta)` 可外部更新并持久化。

**开关**：`ENABLE_IMPORTANCE_WEIGHTED_MEMORY=true`（默认启用）。

### 3.6 Phase C — Extended Thinking / CoT

**文件**：`backend/agent/llm_client.py`、`backend/agent/autonomous_agent.py`

`LLMClient.chat()` 新增 `extra_body` 参数，支持透传任意 body 字段到 OpenAI SDK。

`_generate_action()` 在 `ENABLE_EXTENDED_THINKING=true` 时：
- **Anthropic provider**：注入 `{"thinking": {"type": "enabled", "budget_tokens": <BUDGET>}}`
- **其他 provider**：在 system prompt 前追加 CoT 引导前缀

**开启方式**：
```bash
export MACAGENT_ENABLE_EXTENDED_THINKING=true
export MACAGENT_EXTENDED_THINKING_BUDGET_TOKENS=8000  # 可选调整
```

### 3.8 Skill 按需拉取（On-Demand Skill Fetch）

**文件**：`backend/agent/capsule_on_demand.py`（新建）

#### 背景问题

旧实现（`capsule_strategy.py`）：启动时后台批量同步全部 GitHub 技能（openclaw/5700+、skillcreatorai/47+、anthropics），触发数百次 HTTP 请求，大量下载未必用到的 SKILL.md，浪费带宽与 API 速率。

#### 新设计

| 步骤 | 操作 | 网络请求数 |
|------|------|-----------|
| 启动 | 加载本地 `./capsules/` + 磁盘缓存 + 后台拉取 skill_index（约 10KB）| 1 次 HTTP |
| 任务开始 (`_build_task_guidance`) | 查本地注册表；未命中 → 查 skill_index 匹配名称 → 按需拉取匹配 SKILL.md | 0~3 次 HTTP |
| 结果 | 新技能写磁盘缓存 + 注册入 CapsuleRegistry，下次同任务直接命中 | 0 次 |

#### 技能搜索流程

```
search_and_fetch(task_description)
    ├─ 1. CapsuleRegistry.find_capsule_by_task()  ← 本地，毫秒级
    │       命中 limit → 直接返回
    ├─ 2. skill_index 关键词打分                   ← 内存，毫秒级
    │       top_n 候选技能名
    └─ 3. _fetch_skills_by_names()                ← 网络，秒级（最多 3 个）
            ├─ 检查磁盘缓存 (dist/open_skills/*)
            ├─ 检查内存缓存 (_fetch_cache)
            └─ 按 raw URL 拉取 SKILL.md → parse_skill_md → 注册 + 缓存
```

#### 关键参数

- `search_and_fetch(task, limit=3, min_score=0.5)` — 主入口
- `prefetch_skills_for_task(task)` — fire-and-forget 预拉取
- `MAX_FETCH_PER_REQUEST = 3` — 单次最多 3 个新技能

#### 关闭按需模式（回退全量同步）

```bash
export MACAGENT_ENABLE_ON_DEMAND_SKILL_FETCH=false
```

### 3.9 Benchmark 自动化

**文件**：`backend/scripts/run_benchmark.py`
内置 B1-B7 共 7 个测试用例，通过 WebSocket 连接后端自动执行并汇总结果。

```bash
# 安装依赖
pip install websockets

# 运行所有用例
python3 scripts/run_benchmark.py --url http://localhost:8000

# 运行指定用例
python3 scripts/run_benchmark.py --url http://localhost:8000 --cases B1,B3,B7

# 保存结果
python3 scripts/run_benchmark.py --out ./data/benchmark_results
```

输出：每个用例的成功/失败、耗时、错误信息，以及整体通过率。

## 四、与 v3.1 的关系

v3.2 不破坏 v3.1 任何接口或行为。Phase C 功能均通过 FeatureFlag 控制，可独立关闭退回 v3.1 行为：

```bash
# 回退 Phase C（如需排查）
export MACAGENT_ENABLE_IMPORTANCE_WEIGHTED_MEMORY=false
export MACAGENT_ENABLE_FAILURE_TYPE_REFLECTION=false
export MACAGENT_ENABLE_EXTENDED_THINKING=false
```

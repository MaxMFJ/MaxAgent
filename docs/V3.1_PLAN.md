# v3.1 功能清单（2025–2026 研究/社区共识）

v3.1 在 v3 框架上做「上下文 / 规划 / 升级 / 反思 / 安全 / 目标对齐」增强，不破坏现有主流程。

**实施状态**：v3.1 已落地。见 `app_state` 中 FeatureFlag、`agent/safety.py`、`autonomous_agent` 中 summarize/Goal/Plan/Reflection/Escalation。

---

## 优先级总览（已落地）

| 优先级 | 方向 | 预期收益 |
|--------|------|----------|
| ★★★★★ | 结构化 memory + 上下文压缩 | 显著降低 long-horizon 漂移，token 可控 |
| ★★★★☆ | Plan-and-Execute 分离 | 长任务成功率提升 |
| ★★★★ | Escalation 智能触发 | 更智能 escalation，减少无效循环 |
| ★★★☆ | 反思时机与模型 | 中途纠错、高质量反思降低早停 |
| ★★★ | 危险命令/工具统一校验 | 安全与审计能力提升 |
| ★★☆ | Goal 重述与对齐 | 简单有效防 drift |

上述六项均已实现；详细设计、实施顺序与后续路线见 [主线目标与路线图.md](主线目标与路线图.md)。

---

## 与 v3 的关系

- v3 已提供：错误模型、状态机、并发限流、TimeoutPolicy、config 合并。
- v3.1 在此基础上增加：**上下文与 memory 结构、规划/反思/安全/目标对齐**，不破坏 v3 接口；新逻辑通过 FeatureFlag 或配置开关，便于回滚与 A/B。

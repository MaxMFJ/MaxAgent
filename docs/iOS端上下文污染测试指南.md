## 问题排查

### 如果上下文仍然污染

1. **检查 session_id 是否正确传递：**
```objc
// 在 ChatViewController.m 的 sendMessage 方法中添加日志
NSLog(@"Sending message with session_id: %@", currentConv.conversationId);
```

2. **检查后端日志：**
```bash
# 查看后端是否收到正确的 session_id
tail -f /path/to/backend/logs
```

3. **清空后端缓存：**
```bash
# 删除后端的 session 持久化数据
rm -rf MacAgent/backend/data/contexts/*.json
```

### 如果会话无法保存

1. **检查 UserDefaults：**
```objc
// 在 ConversationManager.m 中添加调试日志
NSLog(@"Saving %lu conversations", (unsigned long)self.conversations.count);
```

2. **验证序列化：**
```objc
// 测试 Conversation 是否正确序列化
NSError *error;
NSData *data = [NSKeyedArchiver archivedDataWithRootObject:conversation 
                                     requiringSecureCoding:YES 
                                                     error:&error];
NSLog(@"Serialization: %@", error ?: @"Success");
```

### 如果 UI 显示异常

1. **检查 currentConversation 是否为空：**
```objc
// 在需要的地方添加断言
NSAssert([ConversationManager sharedManager].currentConversation != nil, @"Current conversation is nil");
```

2. **检查 tableView 数据源：**
```objc
// 在 numberOfRowsInSection 中添加日志
NSLog(@"Table view rows: %lu", (unsigned long)[self currentMessages].count);
```

## 性能优化建议

### 会话列表性能
- 如果会话数量超过 100 个，考虑分页加载
- 可以添加会话归档功能，减少活跃会话数量

### 消息存储优化
- 每个会话的消息超过 500 条时，可以只保存最近的消息
- 老消息可以导出到文件或云端

### 内存优化
- 只在内存中保持当前会话和最近访问的 2-3 个会话
- 其他会话延迟加载

## 与 Mac 端同步

如果需要 Mac 端和 iOS 端共享会话数据，可以考虑：

1. **iCloud 同步**：使用 NSUbiquitousKeyValueStore
2. **服务端存储**：在后端持久化会话数据
3. **导入/导出**：通过文件共享功能

当前版本各端独立管理会话，不会相互同步。

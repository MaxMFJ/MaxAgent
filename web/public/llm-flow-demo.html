大语言模型完整运作原理

Chat 发送「你好」→ 全流程演示 · 物理层标注：当前处理在哪个现实物理设备中完成

🖥️ 用户设备：键盘、CPU、RAM、网卡、显示器 📡 网络：路由器、光纤、基站 🖥️ 服务器：CPU、RAM 🧠 GPU：显存(VRAM)、CUDA 核心 💾 存储：NVMe SSD、PCIe
阶段一 · 用户端 (Client)
1用户输入
你好

用户在输入框键入「你好」，触发发送。

🖥️ 物理层：键盘 → 浏览器进程内存 (RAM)
2前端序列化

将消息封装为 JSON，准备 HTTP/WebSocket 请求。

🖥️ 物理层：用户设备 CPU · 浏览器 JavaScript 引擎 (V8/SpiderMonkey)
3网络发送

请求通过 TCP/HTTP 或 WebSocket 发出。

📡 物理层：用户设备网卡 → 路由器 → 光纤/铜缆/无线基站
阶段二 · 服务端入口
4API 网关 / 负载均衡

请求到达数据中心，由网关路由到推理服务实例。

🖥️ 物理层：云服务器 (如 AWS/GCP/Azure 机房) · 负载均衡器
5请求解析

解析 HTTP 请求体，提取 prompt、参数等。

🖥️ 物理层：推理服务器 CPU · 系统 RAM
阶段三 · 预处理 (Tokenization)
6分词 (Tokenization)

「你好」→ 查词表得到 token IDs，如 [234, 567]。

🖥️ 物理层：CPU · 词表常驻 RAM (或从 SSD 加载)
7上下文组装

拼接 system prompt、历史对话、当前输入，形成完整序列。

🖥️ 物理层：CPU · RAM 中的 token 序列缓冲区
阶段四 · 模型与 Embedding
8模型权重加载

若模型未在显存中，从磁盘加载权重到 GPU VRAM。

💾 物理层：NVMe SSD → PCIe 总线
🧠 物理层：GPU 显存 (HBM/GDDR，如 80GB A100)
9Embedding 查表

Token IDs → 查 Embedding 矩阵 → 得到高维向量 (如 4096 维)。

🧠 物理层：GPU VRAM 中的 Embedding 矩阵 · CUDA 核心执行查表
10位置编码 (Positional Encoding)

为每个 token 注入位置信息，支持模型理解序列顺序。

🧠 物理层：GPU 张量核心 · 矩阵加法
阶段五 · Transformer 推理 (核心)
11Q、K、V 线性投影

输入向量 × Wq, Wk, Wv 得到 Query、Key、Value 矩阵。

🧠 物理层：GPU 矩阵乘法 (GEMM) · Tensor Core
12注意力分数 Q·K^T

计算 Query 与 Key 的点积，得到注意力分数矩阵。

🧠 物理层：GPU 大矩阵乘法 · 高带宽显存读写
13Softmax 归一化

对每行做 Softmax，得到注意力权重分布。

🧠 物理层：GPU 并行 Softmax 核函数
14加权求和 Attention·V

用注意力权重对 Value 加权求和，得到上下文表示。

🧠 物理层：GPU 矩阵乘法 · 多注意力头并行
15FFN 前馈网络

每个 token 经过两层 MLP：Linear → GELU → Linear。

🧠 物理层：GPU 全连接层 · 大量浮点运算
16LM Head + 采样

最后一层输出 → 词表维度 → Softmax → Top-p/Top-k 采样得到下一个 token。

🧠 物理层：GPU 输出层 · 采样在 GPU 完成
阶段六 · 自回归生成
17循环生成 Token

重复步骤 9–16，每次生成一个 token，直到遇到 EOS。

🧠 物理层：GPU 循环推理 · KV Cache 存于显存
阶段七 · 后处理与返回
18Detokenization

Token IDs → 查词表 → 还原为文本「你好！👋 我是...」。

🖥️ 物理层：服务器 CPU · RAM 中的词表
19流式输出

通过 HTTP SSE 或 WebSocket 将文本流式推送给客户端。

📡 物理层：服务器网卡 → 互联网 → 用户设备网卡
20客户端渲染

浏览器接收数据，更新 DOM，渲染消息气泡。

🖥️ 物理层：用户设备 CPU · 浏览器渲染引擎 · 显示器

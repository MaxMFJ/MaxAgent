# Xcode 每次启动都要重新给权限 — 解决方案

## 原因说明

macOS 的 TCC（Transparency, Consent, and Control）是按**应用身份（代码签名）**记录权限的。

- 从 Xcode 直接 Run 时，若未配置 **Development** 签名，Xcode 会使用「仅限本机运行」的临时签名。
- 每次重新编译，临时签名的身份可能变化，系统会认为是**另一个应用**，因此会再次弹出屏幕录制 / 辅助功能等权限。

所以：**用稳定的「开发证书」签名后，同一 Bundle ID 的权限才会在多次构建之间保留。**

---

## 解决步骤（推荐）

### 1. 在 Xcode 里改为「开发」签名

1. 打开 **MacAgentApp.xcodeproj**。
2. 左侧选中 **MacAgentApp** 工程（蓝色图标），再选中 **TARGETS → MacAgentApp**。
3. 顶部切到 **Signing & Capabilities**。
4. 勾选 **Automatically manage signing**（已勾则保持）。
5. 在 **Team** 下拉框中选择你的 **Apple ID / 开发团队**（不能选 “None” 或 “Sign to Run Locally”）。
   - 若没有可选团队：**Xcode → Settings… → Accounts** 添加 Apple ID，再回到这里选 Team。
6. 确认 **Signing Certificate** 显示为 **Apple Development**（或 “Development”），而不是 “Sign to Run Locally”。

这样每次 Debug 构建都会用同一开发身份签名，权限会持久保存。

---

### 2. 清理一次旧的权限记录（可选但建议做一次）

改完签名后建议清理一次 TCC，避免旧「临时签名」记录干扰：

1. **系统设置 → 隐私与安全性**，找到：
   - **屏幕录制**（Screen Recording）
   - 以及若用到了：**辅助功能**（Accessibility）、**自动化**（Automation）
2. 在对应列表中**移除**所有与 **MacAgentApp** 或 **com.macagent.app** 相关的项（若有多个可都删掉）。
3. 关闭设置，回到 Xcode，**重新 Run 一次**，在弹窗中**重新勾选并允许**权限。

之后只要一直用同一 Team 的 Development 签名，从 Xcode 多次 Run 就不会再反复要权限。

---

### 3. 若弹窗是「Python」或「终端」在要权限

若你看到的是 **Python**、**python3** 或 **Terminal** 在请求屏幕录制/辅助功能（因为后端用 `screencapture` 等）：

- 权限是记在 **Python 进程**上的，和 Mac 应用签名无关。
- 在 **系统设置 → 隐私与安全性 → 屏幕录制（及辅助功能）** 里，对 **Python** 或 **Terminal** **勾选并允许一次**即可。
- 确保开发时始终用**同一个 Python**（例如固定用项目 venv 里的 `python`），这样权限会一直有效。

---

## 小结

| 情况 | 做法 |
|------|------|
| 弹窗是 **MacAgentApp** | 在 Xcode 的 Signing & Capabilities 里选好 **Team**，用 Development 签名；必要时在系统设置里删掉旧 MacAgentApp 权限再重新授权一次。 |
| 弹窗是 **Python / Terminal** | 在系统设置里给 **Python** 或 **Terminal** 开一次屏幕录制/辅助功能，并固定使用同一 Python 环境。 |

按上面做完后，再从 Xcode 多次启动 app，权限应不再重复询问。
